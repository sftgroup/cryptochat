Features

How It Works

Three steps to private messaging

1

Connect Your Wallet

Sign a challenge to prove ownership of your address. CryptChat generates an ECDH key pair stored locally in your browser. No private key ever leaves your device.

2

Add a Friend

Search by wallet address or ENS name. When you connect, CryptChat exchanges ECDH public keys and derives a unique shared secret using elliptic curve math.

3

Chat Freely

Every message is encrypted with AES-256-GCM using the derived shared key. The server stores only ciphertext. Only you and your friend can read the messages.

Security

Enterprise-grade encryption by default

CryptChat is designed with a zero-trust security model. Even if the server is compromised, your messages remain unreadable.

🔑 ECDH P-256

NIST-approved elliptic curve key exchange. Each conversation generates a unique 256-bit shared secret.

🛡️ AES-256-GCM

Authenticated encryption with 128-bit integrity tags. Tampered ciphertext is automatically detected and rejected.

📦 Client-Side Only

All cryptographic operations run in your browser using the Web Crypto API. Private keys never touch the network.

🔒 Zero-Knowledge Server

Backend stores only encrypted blobs. No ability to decrypt messages. No key escrow. No backdoors.
